A-Frame Demo

A-Frame is a popular open-source web framework for building virtual reality experiences in the browser.

A-Frame is based on top of HTML to create explorable visual 3D scenes. The language’s core is a powerful entity-component framework that can be programmed extensively by three.js, a JavaScript library used to render 3D graphics.

It was originally developed by the Mozilla VR team in 2015 and it’s currently maintained by developers from Supermedium and they offer ready made component evironments to try out and learn explore.

A-Frame Demo: CDN Call

A-Frame can be developed from a plain .html file. Because we can link directly to the library, there’s no need for downloading any special software and you can get started right away.

In the <head> element, add this line of code:

<script src="https://aframe.io/releases/1.0.4/aframe.min.js"></script>

The <script> element here points to the external script file through the src source attribute.

A-Frame Demo: Primitives

In the <body> element, add an <a-scene> element. The <a-scene> element contains every entity and sets up a 3D scene in the browser.

The A-Frame code is contained inside an <a-scene> element that establishes the scene in the browser. Here, we are using a few basic primitives:

  • <a-box>
  • <a-sphere>
  • <a-cylinder>
  • <a-plane>
  • <a-sky>

A-Frame primitives, such as <a-box>, are a wrapper of larger pieces of code that may easily be reused.

A-Frame Demo: Primitives

Primitives are a great way for beginners to create 3D objects, also known as meshes. They usually have a name that’s easy to remember and come with several bunch of components, such as position, color, width, height, scale.

The button on the bottom-right of the browser is for entering the VR mode with a headset or fullscreen mode on a desktop.

To return to the code editor from the virtual environment, use the esc key or the Menu button on a VR controller.

A-Frame Demo: <a-box>

Use <a-box> just like a normal HTML element, defining the tag and using HTML attributes to customize it. To place a box in our world, put an opening and closing <a-box> tag inside the <a-scene> tag: 


<a-scene>
  <a-box></a-box>
</a-scene>
Want to give it a color: 

<a-scene>
  <a-box color="yellow"></a-box>
</a-scene>

We can give a color value using three different methods:

  • Color names: One of the 140 color names.
  • Hex codes: A six-digit code representing the amount of red, green, and blue for the color.
  • RGB values: A value specified using the rgb() property.

A-Frame Demo: Position

All entities, including the camera (generated by the scene tag), have a position component. And because the default camera and the <a-box> are positioned at the default x, y, z position, 0 0 0, we can only see the box if we move it. Measurements and position are set in meters

    
    <a-scene>
      <a-box  position="2 4 -3"></a-box>
    </a-scene>
A-Frame uses a right-handed coordinate system:

The x-axis value is 2
The y-axis value is 4
The z-axis value is -3

A-Frame Demo: Rotation

Currently, the box looks like a 2D square because it’s facing dead center towards the default camera. To rotate the box, we can change the rotation component:

    
    <a-scene>
      <a-box
        color="red"
        position="0 2 -5"
        rotation="33 45 0">
        <!--
          x-axis rotation - 33°,y-axis - 45°, z-axis - 0°
        -->
      </a-box>
    </a-scene>
To determine the positive direction of rotation, use the right-hand rule. Point our thumbs towards the direction of the positive x, y, or z-axis with your fingers curled:

Positive x-axis goes right.
Positive y-axis goes up.
Positive z-axis goes out of the screen towards us.

The direction in which your fingers curl is the positive direction of rotation. The other way is the negative direction of rotation.

A-Frame Demo: Scale

The scale component defines a shrinking, stretching, or skewing transformation of an entity. It takes three scaling factors for the x, y, and z axes.

  
  <a-scene>
    <a-box
    color="red"
    position="0 2 -5"
    rotation="0 45 45"
    scale="0.5 2 1">
      <!--
        x-axis shrunk by 0.5, y-axis stretched by 3, z-axis unchanged
      -->
    </a-box>
  </a-scene>

All entities inherently have the scale component, so make sure to take advantage of it when designing 3D objects.

A-Frame Demo: Asset Management System

We need to build an environment piece by piece, using different images, still and moving, sounds, and 3D models. We call those assets.

A-frame has an asset management system that makes complex environments with multiple parts load faster. The system makes it easier for the browser to cache assets (e.g., images, videos, 3D models) and A-Frame will make sure all of the assets are fetched before rendering.

To utilize the asset management system, all we need is an <a-assets> element around every HTML element we want to include in the asset management system:

A-Frame Demo: Assets

The asset management system is where we can store textures, audio, and 3D model files inside <a-assets>. There are A-Frame specific components for building a scene:

  • <a-sky> for adding the sky or 360° image.
  • <a-plane> for adding the ground.
  • <a-light> for adding the lighting.
  • <a-sound> for adding the sound.
  • <a-text> for adding text.
  • <a-gltf-model> for adding 3D models.
  • <a-asset-item> also for adding 3D model files

A-Frame also includes animation component for adding animation. And an in-browser tool called the inspector that move around objects. To open the inspector:
control + option + I (Mac)
ctrl + alt + I (Windows)

A-Frame Demo: Assets

Besides the A-Frame specific components called with the <a-assets> tags, one may also include html multimedia tags:

  • <img> for adding the sky or 360° image.
  • <audio> for adding the ground.
  • <video> for adding the lighting.

These assets may be used over and over in a scene.

A-Frame Demo: Assets

To use the asset management system for an image texture:

  
  <a-scene>
    <a-assets>
      <img
        id="stones"
        src="img/sand.jpg" />
    </a-assets>
    <a-box
    src="#stones"
    position="0 2 -5"
    rotation="0 45 45">
    </a-box>
  </a-scene>

Define the texture as an <img>. Give the <img> an id (e.g., id="stones"). And outside <a-assets> reference the asset using the id in selector format (src="#stones").
