Waypoints, NPC Controllers, Audio Mixer

  • Waypoints is defined as an intermediate point on a route or line of travel. If you do not have buildings or barriers in an evironment, you may set up a simple waypoint system for a gameobject without use of the NavMesh system. The demo script is based on this basic waypoints
  • However, if there are obstacles, it's best to create a NavMesh and give the NavMesh agents waypoints as described on this page with the code - Patrol.cs


  • The "manaquin" and animations used in this demos are from Mixamo. Mixamo and Mecanim were introduced on Class 05 - March 3rd. A couple points:
    • The manaquin used in this demo downloaded with the T-Pose as an animation, it appears to be an old file. A way around this is to download character with initial animation - skin and animation.
    • When selecting animations, always turn on In Place before Download
    • Once imported into Unity, select your character and in Materials tab Extract Textures and Materials.
    • Command+D to break animations out of Mixamo object. Rename them and delete the originally imported animations to clean up folder. NOTE: breakoff animation required the Mixamo Avatar

  • The demo had two "Chase" scripts - one for an idle character that comes moves when the player approaches and then attacks at a close proximity. This scipt is titled "Chase.cs" and it is based on Basic AI for NPC
  • The second script sets the non-player character immediately walking to a random direction and then will attack when the player approaches. When the attack is done it sets off in a random direction. This script is based on Unity in Action scripts and the above linked tutorial. It is titled "ChaseAI.cs".
