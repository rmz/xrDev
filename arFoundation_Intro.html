AR Foundation is maintained by Unity and is part of its package registry. AR Foundation enables the construction of Augmented Reality applications targeting both iOS and Android devices. This means you can use AR Foundation to build an AR Unity application and deploy it to either an iOS or Android device without changing any of the Scene's scripts or components.
- from Setting Up AR Foundation

Unfortunately, for the iOS build, there's no way to test from the editor unless one purchases AR Foundation Editor Remote for $80. And my Android is too old to test or run the AR app - the Google Play Services for AR as I get the message: "Your device isn't compatible with this version."

AR Foundation may be employed for vertical and horizontal plane detection, 3D object tracking, face tracking, image tracking as in this brief example. The three packages that will be imported into your project are
    AR Foundation
    ARCore for Android
    ARKit for iOS.

Create a new 3D Core project, be organized with your project files when deciding where to save it. I keep all my Unity projects in one "unityProjects" folder in my Documents folder.

Window > Package Manager - Use the Unity Registry to find and install AR Foundation, ARCore Plugin, ARKit XR Plugin and close window.

From the top menu go to Edit > Project Settings > XR Plug-in Management
    For iOS turn on ARKit
    For Android turn on ARCore
and close window

In the Hierarchy:
1. Delete the Main Camera as an AR specific camera will be used.

2. Add AR Session
The AR Session controls the lifecycle of an AR experience, enabling or disabling AR on the target platform. The ARSession can be on any GameObject. If the Scene doesn't contain an ARSession, the application will not be able to track features in its environment.

3. Add AR Session Origin
This enables virtual object interaction and manipulation with these transformed features.

4. In AR Session, select AR Camera and in its Inspector, change the Tag to MainCamera.

5. Now select AR Sessions Origin from the Hierarchy and in its inspector click Add Component - search Image and select AR Tracked Image Manager.
The AR Tracked Image Manager has:
    Serialized Library - these are the images the app will be looking to detect.
    Max Number of Moving Images - depends on how powerful the device is.
    Tracked Image Prefab - the virtual object that will be instantiated when an image is detected. A prefab is any object that will be repeated in a project or needs to be instantiated by code.

In the Assets folder/window

Create a new folder and call it Prefabs.

Click into the Prefabs folder and right click to open menu of Assets to be generated. Go to Create > XR > Reference Image Library
When the Reference Image Library is selected, the Inspector will show an Add Images folder.

Create a Textures folder and add images to track - .jpg or .png files.

In the Assets folder, create a new folder and name it Textures. Drag and drop from your computer the .jpg or .png files that will be the targets.

Select the Reference Image Library and Add Image. Add your images and turn on Specify Size. The library will recognize the pixel dimensions of the image, but not the game environment or physical size, so set your size, try .02 for the X and the Y will change proportionally. (The size may be determined by the meter scale of the tracked image.)

In the Hierarchy

Select AR Session Origin and from the Assets > Prefab folder drag the Reference Image Library into the Serialized Library or click the target in the field and select it.

In the Hierarchy, create a 3D square or sphere some game object to test the image recognition. Resize it to correspond to the image so .02 for XYZ. Make it into a prefab.

A prefab is intended for game objects that will be repeated in a project and for any game objects that are instantiated by code. For example, a pedestrian in an urban game setting that will show up multiple times and populate the game space via code must be prefabs. Prefabs are generated when game objects are dragged from the Hierarchy into the Assets folder or better into a Prefabs folder in the Assets folder merely to keep your files organized. Once the game object is saved as a prefab in the Assets folder, it can be deleted from the Hierarchy.

In the Build Settings, target appropriate device:

File > Build Settings and select the target device, i.e. Android or iOS and click Switch Platform. (Requires a minute or more, depending on your computer to Import Assets and Build Scripts.)
