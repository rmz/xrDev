Creating a Title Scene

  1. Create a new scene and title simply and clearly
  2. Change the skybox as you wish and add any visuals...
  3. In the Hierarchy create UI > Canvas - this is a 2D element that covers the user's screen
  4. To the Canvas add UI > Text - TextMeshPro as you wish - title, description...
  5. To the Canvas add UI > Button - TextMeshPro and edit as you wish
    1. The Scripts folder has to scripts to change a scene - LoadLevelAfterTime and ChangeScene
    2. In the Hierarchy create an empty game object and label LoadLevel
    3. Add the two scripts to it - LoadLevelAfterTime and ChangeScene
    4. Under - Load Level After Time set the second delay that you wish and type in the name of the scene that should load
  6. Select the button in the Hierarchy and in its Inspector scroll down to the On Click () event, click the plus sign
  7. Drag the LoadLevel game object from the Hierarchy into the field under Runtime Only
  8. From the dropdown adjacent to Runtime Only select ChangeScene.name and again type in the name of the Scene the user is taken to upon click
  9. For the button to work and scene changes to occur, all used scenes must be in the Build Settings - top menu File > Build Settings. Drag and drop your scenes into the Scenes in Build, they should be in order of presentation starting at 0.

Creating, Finding, Editing Textures

As this is not a 3D graphics class, you may create your entire environment using the primitives that Unity provides and decorate them and give them meaning with your own textures. Image dimensions for textures need to be in powers of 2 and are commonly square, i.e 2, 8, 16, 32, 64, 128, 256, 512, 1024... You can have non-square Textures like 512 x 64, just as long as the size remains a power of 2.

Image Resolution, recommended texture formats

Mobile PC
Max 2048
(common for both mobile and PC)		 2048, 4096, 8192, 16384
HDR - high definition render packages commonly uses 4096
recommendations according to graphics processing units (GPS)
My recommendation is to stick with 2048 unless you are targeting high-end computers, then go with 4096. You can always downsize, but you can not upgrade bitmaps without loss of quality

Bump and Displacement Map: Photoshop 3D Filters

Any psd, tiff, jpg, png, gif... (the full list) may be used as a texture in Unity and applied as a material to a 3D object. When you drag a 2d image to say a primitive such as a cube or sphere, Unity automatically generates a material based on the 2d image. Unity automatically flattens multi-layer Photoshop PSD or TIFF files on import so that there is no size penalty for your game. This flattening happens to the imported data in Unity, not to the file itself, so you can continue to save and import your PSD or TIFF files without losing any of your work when using these file types natively. For greater detail add normal and height maps to the material.

Normal Maps use RGB channels to create the illusion of depth. The dark to light values in each the Red, Green and Blue channels are used to have lighter areas pop out whereas dark values recede to create a visual representation of depth.

Bump or Height Maps use luminosity to create the illusion of depth. Bump maps read light and dark areas - luminous values - white creates the illusion of pushing out the pixels - pushing them at your, black the illusion of pushing in the pixels away from you. Although it is a 2D image, light and dark are used to simulate depth or as if it was a 3D object - as if it had depth on the z-index.

The default settings for Blur (0.5) and Detail (80%) tend to work well, but you should experiment with different values and apply them in Unity to achieve the level of detail that you wish

Audio and Video Player

Unity supports these audio file types: .aif, .wav, .mp3, .ogg. As with Photoshop files, Unity will compress the audio, so if you create an aiff or wav file, you should import it rather than compressing it to an mp3.

Unity supports a wide range of video file types, i.e. .avi, .mov, .mp4, .ogv, .webm... However, the optimal codec is H.264. I recommend a high quality mp4 using H.264 codec.
