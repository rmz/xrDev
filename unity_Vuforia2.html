Concerning the project's Hierarchy:
Turning game objects on an off in the inspector allows us to see less information when editing, however, those elements will be part of the final build, so it's important to clean up your hierarchy and delete and game objects that are not used. For example "trump_lp_Model_fbx2014" is from two weeks ago, this is the tpose trump that I originally downloaded. Other Unity notes in general

Concerning the project's Assets:
Although it's necessary to clean up your hierarchy, you do not have to do the same with your Project Assets. The only game objects that end up in the final build are what are in the hierarchy. So you can have a bloated set of assets, but if they are not used in the hierarchy they are not part of the final build. I do however, strongly recommend that you keep a well organized Asset folder.

Other Unity notes in general

How to submit your exercises and projects:

  1. In your Assets > Scene folder, select your project scene (in the example "VuforiaDemo") and then from the top menu go to Assets > Export Package. Post your exported package to your server and link to it from your course homepage.
  2. Build the project and post it to your server for download. For Android, it is an APK
    1. When you are viewing the Scene that you want build go to the top menu > File > Build Settings
    2. In the Build Settings window, make sure that only the Scene that you want built is in the Scenes In Build
    3. Select your Platform, change platform if necessary (this may take a while). Only the platforms that you downloaded with this version of Unity will be available.
    4. As this project uses Vuforia, an AR engine, the platforms would have to be either Android or iOS. Notes for Android
      • Texture Compression Dont' override - ETC
      • ETC2 fallback 32-bit
      • Run Device - a USB connected device should show up here.
      • Compression Method
        None - no or default platform compression.
        LZ4 - fast compression suitable for Development Builds.
        LZ4HC - higher compression rate variance of LZ4, causes longer build times. Works best for Release Builds.

To do so, I duplicated the project in my Finder. I cleaned out all Vufofia content - the Vuforia Scene and all Vuforia folders. I also had to delete the manifest.json and packages-lock.json from my Packages folder through the Finder. (So I cleared out the manifest that still pointed to the Vuforia engine.). Example using SampleScene where I tested game objects that I copied into my Vuforia scene. Select and go to SampleScene. Open Build Settings, add the open scene. Switch Platform to WebGL. Build and Run. I do not save my built projects into the project folder. I maintain a separate folder in my Documents for completed or test builds.
