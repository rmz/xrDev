Physical Material

Unity offers an excellent set of physics algorithms. By adding a Collider component to any game object, you can give it a physics material to adjust friction and bouncing effects of colliding objects. It's important to note that if the "Is Trigger" property is turned on, the collider is used for triggering events and is ignored by the physics engine. However, you can add two colliders - one with "Is Trigger" off and will be effected by the physics of your project and a second to act as a trigger.

By adding a Rigidbody component to any game object, you can give it gravity and act under the control of physics. The Rigidbody can receive forces and torque to make your objects move in a realistic way.

This means that objects to interact with need:

  • A collider such as a Box, Capsule, Sphere or Mesh Collider...
  • Rigidbody

Creating a bouncing ball

  1. In your Assets window right click and select Create > Physics Material. Name it Bounce or Ball...
  2. Change the default properties of the material. Dynamic Friction and Static Friction are by default 0.6, a fraction of 1, because when you multiply fractions you end up with a smaller and smaller number, until it gets to 0 as things come to a hault. If you intend to mimic realism, leave these values at 0.6 and change Bounciness to the same value - 0.6. (If for example it was 1 - 1x1=1 and it would never stop bouncing.)
  3. Dynamic, Static and Bounciness are designed to be kept at a value from 0 to 1.
  4. Friction Combine - Average - two friction values are averaged.
  5. Bounce Combine - Maximum - the largest friction of two values is used.
  6. Apply the new Phyiscal Material to your bouncy game object through it's collider in the Material field.
  7. If you click the target button on the collider Material field, you will see the default surface physical materials of Unity - Carpet, Concrete, Grass, Gravel, Max Friction, Wood, Zero Friction, along with any new Physical Materials that you created.

Picking Up and Throwing Objects

This demo uses Super Character Controller, but broken apart and edited to work with scripts.

Mouse down to pickup object and mouse up drops object, right click to throw object... References: Input and KeyCode

Add empty object to FPC and put it infront of the player - that is where the picked up game object will be visible from the perspective of the player.

Goals of the script:

Playing and Pausing Audio

Unity Audio may be 2D and playing at the same level throughout the environment such as a Narrator that inroduces your project. Or you can have audio be 3D and placed in a specific location or attached to a game object, for example the sound of steps on the FPC controller. You set an Audio Source component to 2D or 3D using the "Spatial Blend" setting.

If it is ambient sound consistently in the background, you can have it Play On Awake and Loop.

You can have the audio triggered by contact with the player or when the player touches another game object that is a trigger. In the second scenario, the Audio Source is a component of a separate game object.

Triggering Audio to Play:

  1. Audio Files always need a game object with an Audio Source Component. Use the Audio Source settings to configure your audio, such as Play On Awake or Loop... For 3D Sound the Min Distance and Max Distance affect the volume.
  2. To trigger the audio, attach the PlayPauseAudio.cs script to the game object that will trigger the audio, whether it's the game object with the Audio Source itself or a separate game object.
  3. The PlayPauseAudio.cs script has two public variables - Audio Interview - place the audio file there and Interview Clip - place the game object with the Audio Source there.
  4. If the script is attached to game object that is to trigger a separate game object with the Audio Source, you do not need to place an audio file in the trigger's Audio Source. (The script always adds an Audio Source by default.)

Playing and Pausing Video:

  1. Create a 3D object to add the video as it's material. If you want the video to play as on a screen use a Plane.
  2. When you drag the video onto the plane, a Video component is automatically added to the game object.
  3. Change the Video Player settings as you wish. Does it Play On Awake? Does it Loop?
  4. Add the PlayPauseVideo.cs script to allow the player to interact with the video.
  5. Create a second plane and position it immediately in front of the video
  6. Give this plane the PlayButton or PauseButton from the Materials folder. In this example, as the PlayButton is meant to be a triangle ontop of the video, there is no pause button or rather visible pause button. I've set this up so that you may add your own play and pause buttons as UI components adjacent to the video
  7. Delete the Mesh Collider on this plane as it will get in the way of using the video itself to play and pause. I turned it off to remember to make this point
  8. The script has three public variables - Play Button Material, Pause Button Material, Screen Renderer. Add the buttons from the Material folder. Drag the PlayButton plane from the Hierarchy to the Screen Renderer field.

Deleting Game Objects

Use the Object.Destroy to delete unnecessary game objects from your game and get better performance.

In the example scene, there is a larger plane below the ground named DestroyPlane in the hierarchy. This plane has the DestroyGameObject.cs script as a component. The script is set up to destroy any game object that collides with it. The Mesh Collider for this plane must have the Convex and Is Trigger turned on.

Commented out in the DestroyGameObject.cs script are alternative uses of the same script file. One is to reset the player at it's starting x,y,z position and destroy all other game objects. And a third option that is commented out is to target game objects with a specific tag. In the Inspector, you may give your game objects a Tag...
